Fix Docker networking, CUDA image tag, and setup script improvements

This commit fixes three critical issues for remote Vista3D setup:

1. Docker Networking for Remote Vista3D (SSH Tunnel)
   - Added extra_hosts mapping in backend/docker-compose.yml
   - Enables Vista3D on Linux to resolve host.docker.internal
   - Works with SSH reverse tunnel (-R 8888:localhost:8888)
   - Fixed VISTA3D_IMAGE_SERVER_URL in setup scripts

2. Fixed Invalid CUDA Image Tag
   - Changed nvidia/cuda:11.8-base-ubuntu22.04 to 11.8.0-base-ubuntu22.04
   - The original tag doesn't exist in Docker Hub
   - Updated in setup.py, backend/setup_backend.py, README.md
   - GPU tests now work correctly

3. Setup Script Improvements
   - Reads existing .env file for NGC_API_KEY and NGC_ORG_ID
   - No longer prompts user if valid credentials exist
   - Fixed KeyError for IMAGE_SERVER_PORT
   - Loads all config values from existing .env when available
